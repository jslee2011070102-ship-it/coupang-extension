# 쿠팡 영상 소재 수집 크롬 확장프로그램 — 클로드 코드 단계별 프롬프트 가이드

> 이 문서는 클로드 코드(Claude Code)에 단계별로 붙여넣을 프롬프트 모음입니다.  
> **반드시 순서대로 진행하고, 각 단계가 정상 동작하는지 확인한 후 다음 단계로 넘어가세요.**

---

## 전체 개발 흐름 요약

```
0단계: 프로젝트 뼈대
  ↓
1단계: 쿠팡 키워드 추출 + 플랫폼 검색 링크 생성
  ↓
2단계: Claude API 대본 생성 + TTS + 자막(SRT) 생성
  ↓
3단계: 썸네일 에디터 (Canvas API)
  ↓
4단계: ZIP 패키징 + 다운로드
  ↓
완성
```

---

## 사전 준비

- **Claude API 키**: https://console.anthropic.com 에서 발급
- **작업 폴더**: 바탕화면 등 원하는 위치에 `coupang-extension` 폴더 생성
- **클로드 코드 실행**: 해당 폴더에서 Claude Code 실행

---

## 0단계 — 프로젝트 뼈대 만들기

> 예상 소요 시간: 30분  
> 목표: 크롬에 로드할 수 있는 빈 확장프로그램 기본 구조 생성

### 클로드 코드에 붙여넣을 프롬프트

```
크롬 확장프로그램을 개발할 거야.

기능 개요:
쿠팡에서 제품 검색 시 URL에서 키워드를 자동 추출하고,
관련 영상 소재를 수집한 뒤, AI 대본/TTS/자막/썸네일을 만들어서
ZIP으로 패키징해주는 크롬 확장프로그램이야.

지금은 0단계로, 기본 파일 구조만 만들어줘.

생성할 파일 목록:
- manifest.json (Manifest V3 기준)
- popup.html (기본 UI 뼈대, 나중에 기능 붙일 수 있게 섹션 구분만 해줘)
- popup.js (빈 파일, 주석으로 나중에 뭘 채울지만 표시)
- content.js (빈 파일, 주석으로 역할 설명)
- styles.css (기본 스타일만, 확장프로그램 팝업 크기 360px x 600px 기준)

요청사항:
1. 각 파일 역할을 코드 상단 주석으로 설명해줘
2. manifest.json에 필요한 권한(tabs, activeTab, storage, scripting)도 포함해줘
3. 크롬에서 로컬 테스트 로드하는 방법 단계별로 알려줘
4. 폴더 구조도 트리 형태로 보여줘
```

### 0단계 완료 확인 체크리스트

- [ ] 파일 4개가 정상 생성되었는가
- [ ] 크롬 → 확장프로그램 → 개발자 모드 → 압축해제된 확장 프로그램 로드 → 폴더 선택
- [ ] 크롬 툴바에 확장프로그램 아이콘이 나타나는가
- [ ] 아이콘 클릭 시 빈 팝업이 열리는가

## 1단계 — 쿠팡 키워드 추출 + 검색 링크 생성

> 예상 소요 시간: 2~3시간  
> 목표: 쿠팡 페이지에서 제품명 키워드를 자동 추출하고, 각 플랫폼 검색 링크를 팝업에 표시

### 클로드 코드에 붙여넣을 프롬프트

```
1단계 기능을 추가해줘.

[키워드 추출]
쿠팡(coupang.com) 제품 페이지를 열었을 때
content.js가 URL 또는 페이지 제목(h1 태그)에서 제품명 키워드를 자동 추출해서
팝업(popup.js)으로 전달해줘.

URL 예시: https://www.coupang.com/vp/products/123456?itemId=...&q=무선청소기
→ 추출 키워드: "무선청소기"

[번역]
추출된 키워드를 Google Translate 무료 API를 사용해서
영문과 중문으로 각각 번역해줘.

번역 결과는 별도로 강조할 필요 없고,
검색 링크 위에 작은 텍스트로 "검색 키워드: wireless vacuum cleaner / 无线吸尘器" 형태로
확인용으로만 조용히 표시해줘.
사용자가 검색 링크를 누르기 전에 어떤 키워드로 검색되는지 확인할 수 있는 정도면 충분해.

[검색 링크 생성]
번역된 키워드(영문/중문)로 아래 플랫폼 검색 링크를 각각 생성해서
팝업에 목록으로 표시해줘:
- 틱톡 (tiktok.com/search?q=키워드)
- 도우인 (douyin.com/search/키워드)
- 알리익스프레스 (aliexpress.com/wholesale?SearchText=키워드)
- 인스타그램 (instagram.com/explore/tags/키워드)

각 링크는 클릭 시 새 탭으로 열리게 해줘.
영문 검색결과 / 중문 검색결과를 구분해서 표시해줘.

[UI]
팝업 상단에 추출된 키워드와 번역 결과를 보여주고,
하단에 플랫폼 링크 목록을 보여줘.
쿠팡 페이지가 아닌 다른 사이트에서 열면 "쿠팡 상품 페이지에서 실행해주세요" 안내 문구를 표시해줘.
```

### 1단계 완료 확인 체크리스트

- [ ] 쿠팡 상품 페이지에서 확장프로그램 아이콘 클릭
- [ ] 제품명 키워드가 자동 추출되어 팝업에 표시되는가
- [ ] 영문/중문 번역 결과가 표시되는가
- [ ] 각 플랫폼 링크 클릭 시 새 탭으로 검색 결과가 열리는가
- [ ] 쿠팡이 아닌 페이지에서 안내 문구가 표시되는가

## 2단계 — AI 대본 생성 + TTS + 자막(SRT) 생성

> 예상 소요 시간: 2~3시간  
> 목표: 사용자가 입력한 스토리라인을 Claude API로 대본화하고, TTS 음성과 SRT 자막 파일 생성  
> 필요한 것: Claude API 키 (https://console.anthropic.com)

### 클로드 코드에 붙여넣을 프롬프트

```
2단계 기능을 추가해줘.

[설정 화면]
팝업에 설정(⚙) 버튼을 추가해줘.
설정 화면에서 Claude API 키를 입력하고 저장할 수 있게 해줘.
API 키는 chrome.storage.local에 저장해줘.
저장된 API 키는 마스킹(****) 처리해서 표시해줘.

[대본 생성]
팝업에 텍스트 입력창을 추가해줘.
입력창에 제품 스토리라인/핵심 소구포인트를 입력하면
Claude API (claude-sonnet-4-20250514 모델)를 호출해서
30초~1분 분량의 쇼츠 대본을 생성해줘.

Claude API 호출 시 시스템 프롬프트:
"당신은 쇼츠 영상 대본 전문가입니다. 
입력된 제품 소구포인트를 바탕으로 
30초~1분 분량(150~250자)의 자연스럽고 흥미로운 한국어 쇼츠 대본을 작성하세요.
대본은 문장 단위로 줄바꿈 하여 구분해주세요."

생성된 대본은 팝업에 표시하고 수정 가능하게 해줘.

[TTS 생성]
대본 생성 후 "음성 생성" 버튼을 누르면
Web Speech API (SpeechSynthesis)로 TTS 음성을 생성해줘.

중요: TTS 로직은 반드시 별도 파일 tts.js로 분리해줘.
나중에 다른 TTS 서비스(Google Cloud TTS, ElevenLabs 등)로
교체하기 쉽도록 아래 인터페이스 형태로 만들어줘:

tts.js 인터페이스:
- generateTTS(text): 음성 생성 후 MP3 Blob 반환
- getTimings(text): 각 문장별 시작/종료 시간(ms) 배열 반환

생성된 음성은 팝업에서 미리 들을 수 있게 재생 버튼도 추가해줘.
MP3 파일로 저장할 수 있게 다운로드 버튼도 추가해줘.

[자막(SRT) 생성]
TTS 생성 시 getTimings()로 각 문장의 타이밍을 추출해서
SRT 형식의 자막 파일을 자동 생성해줘.

SRT 형식 예시:
1
00:00:00,000 --> 00:00:03,500
이 제품 하나로 피부가 달라졌어요

2
00:00:03,500 --> 00:00:07,000
직접 써보고 진짜 놀랐는데

생성된 SRT는 팝업에서 미리보기 가능하게 하고
파일로 다운로드도 가능하게 해줘.

[진행 상태]
API 호출 중에는 로딩 스피너와 "대본 생성 중..." 텍스트를 표시해줘.
에러 발생 시(API 키 없음, 네트워크 오류 등) 친절한 안내 메시지를 표시해줘.
```

### 2단계 완료 확인 체크리스트

- [ ] 설정 화면에서 API 키 저장이 되는가
- [ ] 스토리라인 입력 후 "대본 생성" 버튼 클릭 시 대본이 생성되는가
- [ ] 생성된 대본이 수정 가능한가
- [ ] "음성 생성" 버튼 클릭 시 TTS 음성이 생성되는가
- [ ] 생성된 음성을 팝업에서 미리 들을 수 있는가
- [ ] SRT 자막 파일이 생성되는가
- [ ] tts.js 파일이 별도로 분리되어 있는가

## 3단계 — 썸네일 에디터

> 예상 소요 시간: 3~4시간  
> 목표: 썰채널 형식의 썸네일을 팝업 안에서 직접 만들고 PNG로 저장

### 클로드 코드에 붙여넣을 프롬프트

```
3단계 기능을 추가해줘.

[썸네일 에디터 UI]
팝업에 썸네일 에디터 탭을 추가해줘.
썰채널 형식 레이아웃:
- 상단 박스: 메인 제목 입력 (큰 글씨, 눈에 띄는 색상)
- 하단 영역: 부제목/설명 입력 (작은 글씨)

편집 옵션:
- 제목 폰트 선택 (기본 3가지: 고딕/명조/손글씨 느낌)
- 제목 글자 크기 조절 (슬라이더)
- 제목 글자 색상 선택 (컬러피커)
- 상단 박스 배경색 선택 (컬러피커)
- 전체 배경색 선택 (컬러피커)

[실시간 미리보기]
편집 내용이 변경될 때마다 팝업 안에 썸네일 미리보기가 실시간으로 업데이트되게 해줘.
미리보기 캔버스 크기: 1280x720 (유튜브 썸네일 표준) 비율로 팝업 안에 맞게 축소 표시

[이미지 저장]
"썸네일 저장" 버튼 클릭 시
HTML Canvas API로 1280x720 PNG 이미지를 렌더링해서 다운로드해줘.
파일명: thumbnail_날짜시간.png

[다음 단계 연동 준비]
생성된 썸네일 이미지 Blob을 나중에 4단계 ZIP 패키징에서 사용할 수 있도록
변수에 저장해두는 구조로 만들어줘.
```

### 3단계 완료 확인 체크리스트

- [ ] 썸네일 에디터 탭이 팝업에 표시되는가
- [ ] 제목/부제목 입력 시 미리보기가 실시간으로 바뀌는가
- [ ] 폰트/크기/색상 변경이 미리보기에 즉시 반영되는가
- [ ] "썸네일 저장" 버튼 클릭 시 PNG 파일이 다운로드되는가
- [ ] 다운로드된 이미지가 1280x720 해상도인가

## 4단계 — ZIP 패키징 + 다운로드

> 예상 소요 시간: 1~2시간  
> 목표: 1~3단계에서 만든 모든 결과물을 ZIP으로 묶어서 한 번에 다운로드

### 클로드 코드에 붙여넣을 프롬프트

```
4단계 기능을 추가해줘.

[ZIP 패키징]
JSZip 라이브러리를 사용해서
지금까지 만든 모든 결과물을 ZIP으로 묶어서 다운로드하는 기능을 추가해줘.

ZIP에 포함할 파일:
1. script.txt — 2단계에서 생성된 AI 대본 (UTF-8 텍스트)
2. tts_audio.mp3 — 2단계에서 생성된 TTS 음성
3. subtitle.srt — 2단계에서 생성된 SRT 자막
4. thumbnail.png — 3단계에서 생성된 썸네일 이미지
5. order.txt — 캡컷 편집 순서 안내 텍스트 (아래 내용 포함)

order.txt 내용 예시:
---
[캡컷 편집 순서 안내]
1. thumbnail.png → 첫 장면 (2~3초)
2. 수집한 제품 영상들 (순서대로 배치)
3. tts_audio.mp3 → 전체 오디오 트랙으로 삽입
4. subtitle.srt → 자막 불러오기로 삽입

[편집 팁]
- 캡컷 → 새 프로젝트 → 9:16 세로형 선택
- 오디오 길이에 맞게 영상 클립 길이 조절
- 자막은 상단 메뉴 → 텍스트 → 자막 불러오기
---

[ZIP 파일명]
키워드_YYYYMMDD_HHMMSS.zip 형식으로 저장

[UI]
팝업에 "📦 소재 패키징 다운로드" 버튼 추가해줘.
버튼 클릭 전에 완성된 항목 체크리스트를 표시해줘:
- ✅ 대본 생성 완료
- ✅ TTS 음성 생성 완료  
- ✅ 자막 생성 완료
- ✅ 썸네일 생성 완료
- ⬜ 미완성 항목은 회색으로 표시

미완성 항목이 있으면 경고 메시지 표시하되, 완성된 것만으로도 다운로드 가능하게 해줘.
ZIP 생성 중에는 로딩 표시를 해줘.

[JSZip 설치]
npm 대신 CDN으로 불러와줘:
<script src="https://cdnjs.cloudflare.com/ajax/libs/jszip/3.10.1/jszip.min.js"></script>
```

### 4단계 완료 확인 체크리스트

- [ ] "소재 패키징 다운로드" 버튼이 팝업에 표시되는가
- [ ] 완성 항목 체크리스트가 올바르게 표시되는가

## 막혔을 때 사용할 프롬프트


## 나중에 업그레이드할 때 사용할 프롬프트

### TTS를 유료 서비스로 교체할 때

```
현재 Web Speech API로 구현된 tts.js를
Google Cloud TTS API로 교체해줘.

기존 인터페이스 유지:
- generateTTS(text): 음성 생성 후 MP3 Blob 반환
- getTimings(text): 각 문장별 시작/종료 시간(ms) 배열 반환

Google Cloud TTS API 키는 설정 화면에서 입력하고 저장할 수 있게 해줘.
```

### 영상 합치기 자동화로 업그레이드할 때

```
현재 ZIP 다운로드 방식을 Creatomate API를 사용한 자동 영상 합치기로 업그레이드해줘.
ZIP 안의 소재들을 Creatomate API로 전송해서 MP4를 자동으로 생성하고 다운로드하게 해줘.
Creatomate API 키는 설정 화면에서 입력하고 저장할 수 있게 해줘.
```
